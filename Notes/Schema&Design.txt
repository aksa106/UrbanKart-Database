
PROJECT : UrbanKart



UrbanKart is a fictional ecommerce company created
to practice real-world database design and SQL analysis.

This project simulates how ecommerce businesses store,
manage, and analyze customer, product, and order data
using Microsoft SQL Server.


-------------------------------------------------------------------


TABLE DESIGN & SCHEMA-------------------


1. CUSTOMERS TABLE

-The Customer table just modelling for basic account information for users signing up on the UrbanKart ecommerce platform

-Just - identity, contact info, account creation tracking


SCHEMA -- Customers(CustomerID, FirstName, LastName, Email, Phone, CreatedAt)


----------------------------------

2. PRODUCT TABLE

-store all of the sellable products listed on UrbanKart

SCHEMA -- Products(ProductID, ProductName, Category,  Price, StockQty)

ProductID - uniquely identifies every product in the system, it can be used for inventory tracking, Orders, relationship with other tables 

Category - Groups products into business categories , useful for, filtering , analytics, dashboards, recommendations 

Price - we chose DECIMAL datatype for price because money should store precise values FLOAT can create rounding inaccuracies in financial data.
----------------------------------------

3.ORders TABLE

The Orders table stores all purchase transactions made on the UrbanKart ecommerce platform.
It connects customers and products to track who bought what, in what quantity, and when.
This table is the foundation for sales analysis, revenue tracking, and order history.

SCHEMA -- Orders (OrderID, CustomerID, ProductID, Quantity, TotalAmount, OrderDate)


Foreign key:  -- referential integrity
cutomer table - customerID - Primary key , customer will always exist
ORders table - customerID refrences CustomerID of Customers table 

FOREIGN KEY(COLUMN_NAME),    [same column in both tables]
REFERENCES TABLE_NAME(COLUMN_NAME)  [table name - table we are connecting our current table to]
